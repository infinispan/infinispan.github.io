# Issue Creation Instructions

## Issue Types

The website repository does not have issue templates. Use plain GitHub issues with a descriptive title and body.

Common issue categories:
- **Content** — blog post requests, page updates, broken links, outdated information
- **Styling** — visual bugs, responsive layout issues, CSS fixes
- **Infrastructure** — build failures, plugin issues, deployment problems, dependency updates
- **Release** — new version additions, download link updates, release notes

## Conventions

- Issue titles should be concise and descriptive
- Reference related issues and PRs using `#number` format
- For visual bugs, include screenshots and the browser/viewport where the issue occurs
- For build issues, include the error output and Ruby/Jekyll version
- Commit messages referencing issues use the format: `Fix issue description (#NNN)`
