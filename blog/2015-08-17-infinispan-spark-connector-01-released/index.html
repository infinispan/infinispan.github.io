Infinispan Spark connector 0.1 released!

<p><div dir="ltr" style="text-align: left;" trbidi="on">Dear users, <br/><br/>The Infinispan connector for Apache Spark has just been made available as a <a href="http://spark-packages.org/package/infinispan/infinispan-spark">Spark Package</a>!<br/><br/><h3 style="text-align: left;">What is it?</h3><br/>The Infinispan Spark connector allows tight integration with Apache Spark, allowing Spark jobs to be run against data stored in the Infinispan Server, exposing any cache as an RDD, and also writing data from any key/value RDD to a cache. It's also possible to create a DStream backed by cache events and to save any key-value DStream to a cache.<br/><br/>The minimum version required is Infinispan 8.0.0.Beta3.<br/><br/><h3 style="text-align: left;">Giving it a spin with Docker</h3><br/>A handy docker image that contains an Infinispan cluster co-located with an Apache Spark standalone cluster is the fastest way to try the connector. Start by launching the container that hosts the Spark Master:<br/><br/><script src="https://gist.github.com/7ea2bc8aaf9e84e4f91b.js"> </script><br/>And then run as many worker nodes as you want:<br/><br/><script src="https://gist.github.com/db47ec0d8208ba3391a7.js"> </script><br/><h3 style="text-align: left;">Using the shell</h3><br/>The Apache Spark shell is a convenient way to quickly run jobs in an interactive fashion. Taking advantage of the fact that Spark is already installed in the docker containers (and thus the shell), let's attach to the master:<br/><br/><script src="https://gist.github.com/00cd2c4bd34baf130f9c.js"> </script><br/>Once inside, a Spark shell can be launched by:<br/><br/><script src="https://gist.github.com/bb9f1f7c7a5982042bbb.js"> </script><br/>That's all it's needed. The shell grabs the Infinispan connector and its dependencies from spark-packages.org and exposes them in the classpath.<br/><br/><h3 style="text-align: left;">Generating data and writing to Infinispan</h3><br/>Let's obtain a list of words from the Linux dictionary, and generate 1k random 4-word phrases. Paste the commands in the shell:<br/><br/><script src="https://gist.github.com/18b9b4fb216d96a1a34b.js"> </script><br/>From the phrases, we'll create a key value RDD (Long, String):<br/><br/><script src="https://gist.github.com/dd5bb83113bb42399af0.js"> </script><br/>To save to Infinispan:<br/><br/><script src="https://gist.github.com/cbe858fdd3e669fac056.js"> </script><br/><br/><h3 style="text-align: left;">Obtaining facts about data</h3><br/>To be able to explore data in the cache, the first step is to create an infinispan RDD:<br/><br/><script src="https://gist.github.com/0de646aee6e81c524878.js"> </script><br/>As an example job, let's calculate a histogram showing the distribution of word lengths in the phrases. This is simply a sequence of transformations expressed by:<br/><br/><script src="https://gist.github.com/3787787931e3c5f9d082.js"> </script><br/>This pipeline yields:<br/><br/>2 chars words: 10 occurrences<br/>3 chars words: 37 occurrences<br/>4 chars words: 133 occurrences<br/>5 chars words: 219 occurrences<br/>6 chars words: 373 occurrences<br/>7 chars words: 428 occurrences<br/>8 chars words: 510 occurrences<br/>9 chars words: 508 occurrences<br/>10 chars words: 471 occurrences<br/>11 chars words: 380 occurrences<br/>12 chars words: 309 occurrences<br/>13 chars words: 238 occurrences<br/><br/>… <br/><br/>Now let's find similar words using the <a href="https://en.wikipedia.org/wiki/Levenshtein_distance">Levenshtein distance algorithm</a>. For that we need to define a function that will calculate the edit distance between two strings. As usual, paste in the shell:<br/><br/><br/><script src="https://gist.github.com/9e9430507a40fd3f410b.js"> </script><br/>Empowered by the Levenshtein distance implementation, we need another function that given a word, will find in the cache similar words according to the provided maximum edit distance:<br/><br/><script src="https://gist.github.com/9d31ba5cc650f2a050fd.js"> </script><br/>Sample usage:<br/><br/><script src="https://gist.github.com/f3e6b342b1eae478feb5.js"> </script><br/><h3 style="text-align: left;">Where to go from here</h3><br/>And that concludes this first post on Infinispan-Spark integration.
