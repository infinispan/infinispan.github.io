Monday 03 April 2017

Infinispan Spark connector 0.5 released!

<p><div dir="ltr" style="text-align: left;" trbidi="on">The Infinispan Spark connector offers seamless integration between Apache Spark and Infinispan Servers.<br/>Apart from supporting Infinispan 9.0.0.Final and Spark 2.1.0, this release brings many usability improvements, and support for another major Spark API.<br/><br/><h3 style="text-align: left;">Configuration changes</h3><br/>The connector no longer uses a <i>java.util.Properties</i> object to hold configuration, that's now duty of<i> org.infinispan.spark.config.ConnectorConfiguration</i>, type safe and both Java and Scala friendly:<br/><br/><script src="https://gist.github.com/gustavonalle/8007b0fee8f5ef7eaf96f45912970391.js"/> <br/><h3 style="text-align: left;"/><h3 style="text-align: left;">Filtering by query String</h3><br/>The previous version introduced the possibility of filtering an InfinispanRDD by providing a <i>Query</i> instance, that required going through the QueryDSL which in turn required a properly configured remote cache. <br/><br/>It's now possible to simply use an Ickle query string:<br/><br/><script src="https://gist.github.com/gustavonalle/e17322a5ae6547a6ab2746ac8732f4c0.js"/> <br/><br/><h3 style="text-align: left;">Improved support for Protocol Buffers</h3><br/>Support for reading from a Cache with protobuf encoding was present in the previous connector version, but now it's possible to also write using protobuf encoding and also have protobuf schema registration automatically handled.<br/><br/>To see this in practice, consider an arbitrary non-Infinispan based<i> RDDInteger, Hotel</i> where Hotel is given by:<br/><br/><script src="https://gist.github.com/gustavonalle/97e2f4ebdb51454ecb2587f0e9f92944.js"/> <br/>In order to write this RDD to Infinispan it's just a matter of doing:<br/><br/><script src="https://gist.github.com/gustavonalle/b1def1a094ea138eec892c8c843c06e3.js"/> Internally the connector will trigger the auto-generation of the <i>.proto</i> file and message marshallers related to the configured entity(ies) and will handle registration of schemas in the server prior to writing. <br/><br/><br/><br/><h3 style="text-align: left;">Splitter is now pluggable</h3><br/>The <i>Splitter</i> is the interface responsible to create one or more partitions from a Infinispan cache, being each partition related to one or more segments. The Infinispan Spark connector now can be created using a custom implementation of Splitter allowing for different data partitioning strategies during the job processing.<br/><br/><br/><h3 style="text-align: left;">Goodbye Scala 2.10</h3><br/>Scala 2.10 support was removed, Scala 2.11 is currently the only supported version. Scala 2.12 support will follow <a href="https://issues.apache.org/jira/browse/SPARK-14220">https://issues.apache.org/jira/browse/SPARK-14220</a><br/><br/><br/><h3 style="text-align: left;"/><h3 style="text-align: left;">Streams with initial state</h3><br/>It is possible to configure the <i><span class="pl-en">InfinispanInputDStream</span></i> with an extra boolean parameter to receive the current cache state as events.<br/><br/><h3 style="text-align: left;"/><h3 style="text-align: left;">Dataset support</h3><br/>The Infinispan Spark connector now ships with support for Spark's Dataset API, with support for pushing down predicates, similar to <i>rdd.filterByQuery</i>. The entry point of this API is the Spark session:<br/><br/><script src="https://gist.github.com/gustavonalle/c794b2e9c7ed1cb20a9c97b193df8c75.js"/> <br/>To create an Infinispan based Dataframe, the "infinispan" data source need to be used, along with the usual connector configuration:<br/><br/><script src="https://gist.github.com/gustavonalle/5a409cfd64f27a2d2d2b92971c1f60af.js"/> From here it's possible to use the untyped API, for example:<br/><br/><script src="https://gist.github.com/gustavonalle/f22f00ea02d2c85bb806ff4cd82f7ded.js"/> or execute SQL queries by setting a view:<br/><br/><script src="https://gist.github.com/gustavonalle/2203b3694c14f585bd9107f0e9b44133.js"/> In both cases above, the predicates and the required columns will be converted to an Infinispan Ickle filter, thus filtering data at the source rather than at Spark processing phase.<br/><br/><br/>For the full list of changes see the <a href="https://issues.jboss.org/secure/ReleaseNote.jspa?projectId=12316820&amp;version=12331257">release notes.</a> For more information about the connector, the <a href="https://github.com/infinispan/infinispan-spark/blob/master/README.md">official documentation</a> is the place to go. Also check the <a href="https://github.com/infinispan/infinispan-spark/tree/master/examples/twitter">twitter data processing sample</a> and to report bugs or request new features use the <a href="https://issues.jboss.org/projects/ISPRK">project JIRA</a>.<br/><br/><br/><br/></div></p>

Posted by Gustavo Fernandes
