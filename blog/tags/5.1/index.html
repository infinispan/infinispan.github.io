Infinispan 5.1.4.FINAL is out now!

Infinispan 5.1.4.FINAL is out now fixing a bunch of issues as well as enhancing performance of Infinispan’s Memcached and REST servers, plus it now exposes a couple of new JMX attributes such as cache view (RpcManager.CommittedViewAsString &Â RpcManager.PendingViewAsString) and cluster name.

JDK 8 backported ConcurrentHashMaps in Infinispan

Doug Lea and the folks on the concurrency-interest group have been hard at work on an update of JSR 166Â (concurrency utilities)Â for Java 8 - called JSR 166e. Â These include some pretty impressive changes in a building-block we’ve all come to rely pretty heavily on, the ConcurrentHashMap.

One if the big drawbacks in the ConcurrentHashMap, since it was introduced in Java 5, has always been memory footprint. Â It is kinda bulky, especially when compared to a regular HashMap - 1.6 kb in memory versus about 100 bytes! Â Yes, these are for emptyÂ maps.

In Java 8, one of the improvements in theÂ ConcurrentHashMapÂ has been memory footprint - now closer to a regularÂ HashMap. Â In addition to that, the new Java 8 CHM performs better under concurrent load when compared to its original form. Â See this discussionÂ and comments in the proposed ConcurrentHashMapV8 sources for more details.

So, Infinispan makes pretty heavy use ofÂ ConcurrentHashMaps internally. Â One change in the recently released Infinispan 5.1.2.Final is these internal CHMs are built using a factory, and we’ve included a version of the Java 8 CHM in Infinispan. Â So by default, Infinispan uses the JDK’s provided CHM. Â But if you wish to force Infinispan to use the backported Java 8 CHM, all you need to do is include the following JVM parameter when you start Infinispan:

-Dinfinispan.unsafe.allow_jdk8_chm=true

We’d love to hear what you have to say about this, in terms of memory footprint, garbage collection and overall performance. Â Please use the Infinispan user forums to provide your feedback.

Monday, 27 February 2012

Infinispan 5.1.2.CR1 is out!

Based on community demand, we’ve decided to do another minor release in the Infinispan 5.1 'Brahma' series and we’ve just released Infinispan 5.1.2.CR1.

This release includes fixes in the generated documentation, schema, and by adding remote JMX monitoring options for all our demos, we’ve made it easier to consume Infinispan out of the box.

Some important library upgrades such as JGroups have also been upgraded and we’ve implemented more performance improvements, particularly around state transfer.

Finally, we’ve added a new system property (-Dinfinispan.unsafe.allow_jdk8_chm=true) which can be used to replace internal Java’s concurrent ConcurrentHashMap with the forthcoming JDK8Â ConcurrentHashMapV8 which is more memory efficient than the currentÂ ConcurrentHashMap.

Full details of what has been fixed can be found here, and if you have feedback, please visit our forums. Finally, as always, you can download the release from here.

Cheers, Galder
