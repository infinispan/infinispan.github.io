Reactive Big Data on OpenShift In-Memory Data Grids

<p><div dir="ltr" style="text-align: left;" trbidi="on">Thanks a lot to everyone who attended the Infinispan sessions I gave in <a href="http://www.developermarch.com/developersummit/">Great Indian Developer Summit</a>! Your questions after the talks were really insightful.<br/><div><br/></div><div>One of the talks I gave was titled <a href="http://www.developermarch.com/developersummit/session.html?insert=Galder1">Big Data In Action with Infinispan</a>(slides are available <a href="https://speakerdeck.com/galderz/big-data-in-action-with-infinispan">here</a>), where I was looking at how Infinispan based in-memory data grids can help you deal with the problems of real-time big data and how to do big data analytics.</div><div><br/></div><div>During the talk I live coded a demo showing both real-time and analytics parts, running on top of <a href="https://www.openshift.org/">OpenShift</a> and using <a href="http://vertx.io/">Vert.x</a> for joining the different parts. The demo repository contains background information on <a href="https://github.com/galderz/swiss-transport-datagrid/blob/master/README.md">instructions</a> to get started with the demo, but I thought it'd be useful to get focused step-by-step instructions in this blog post.</div><div><br/></div><h2 style="text-align: left;">Set Up</h2><div><br/></div><div><div>Before we start with any of the demos, it's necessary to run some set up steps:</div></div><div><br/></div><div> 1. Check out git repository:</div><div> </div><div><div> <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">git clone https://github.com/galderz/swiss-transport-datagrid</span></div></div><div><br/></div><div> 2. Install <a href="https://www.openshift.org/">OpenShift Origin</a> or <a href="https://www.openshift.org/minishift/">Minishift</a> to get an OpenShift environment running in your own</div><div> machine. I decided to use OpenShift Origin, so the instructions below are tailored for that</div><div> environment, but similar instructions could be used with Minishift.</div><div><br/></div><div> 3.Install <a href="https://www.continuum.io/downloads">Anaconda for Python 3</a>, this is required to run Jupyter notebook for plotting.</div><div><br/></div><h2 style="text-align: left;">Demo Domain</h2><div><br/></div><div>Once the set up is complete, it's time to talk about the demos before we run them.</div><div><br/></div><div>Both demos shown below work with the same application domain: swiss rail transport systems. In this domain, we differentiate between physical stations, trains, station boards which are located in stations, and finally stops, which are individual entries in station boards.</div><div><br/></div><h2 style="text-align: left;">Real Time Demo</h2><div><br/></div><div>The first demo is about working with real-time data from station boards around the country and presenting a centralised dashboard of delayed trains around the country. The following diagrams shows how the following components interact with each other to achieve this:</div><div><br/><div class="separator" style="clear: both; text-align: center;"><a href="https://3.bp.blogspot.com/-mu5jN_vRnVk/WQxMd3Ev8aI/AAAAAAAAE2g/R2hUTM9_d4U3sPWsmKl8VyEwoLPwR-VhwCLcB/s1600/real-time-demo.jpg" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" height="268" src="https://3.bp.blogspot.com/-mu5jN_vRnVk/WQxMd3Ev8aI/AAAAAAAAE2g/R2hUTM9_d4U3sPWsmKl8VyEwoLPwR-VhwCLcB/s640/real-time-demo.jpg" width="640"/></a></div><br/></div><div/><div><br/></div><div>Infinispan, which provides the in-memory data grid storage, and Vert.x, which provides the glue for the centralised delayed dashboard to interact with Infinispan, all run within OpenShift cloud.</div><div><br/></div><div>Within the cloud, the Injector verticle cycles through station board data and injects it into Infinispan. Also within the cloud, a Vert.x verticle that uses <a href="http://infinispan.org/docs/stable/user_guide/user_guide.html#query.continuous">Infinispan's Continuous Query</a> to listen for station board entries that are delayed, and these are pushed into the Vert.x event bus, which in turn, via a <a href="http://vertx.io/docs/vertx-web/java/#_sockjs_event_bus_bridge">SockJS bridge</a>, get consumed via WebSockets from the dashboard. The centralised dashboards is written with <a href="http://docs.oracle.com/javase/8/javafx/get-started-tutorial/jfx-overview.htm#JFXST784">JavaFX</a> and runs outside the cloud.</div><div><br/></div><div>To run the demo, do the following:</div><div><br/></div><div> 1. Start OpenShift Origin if you've not already done so:</div><div><br/></div><div> <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">oc cluster up –public-hostname=127.0.0.1</span></div><div><br/></div><div> 2. Deploy all the OpenShift cloud components:</div><div><br/></div><div> <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">cd ~/swiss-transport-datagrid</span></div><div> <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">./deploy-all.sh</span></div><div><br/></div><div> 3. Open the <a href="https://127.0.0.1:8443/console/project/myproject/overview">OpenShift console</a> and verify that all pods are up.</div><div><br/></div><div> 4. Load github repository into your favourite IDE and run</div><div> <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;"><b>delays.query.continuous.fx.FxApp</b></span> Java FX application. This will load the</div><div> centralised dashboard. Within seconds delayed trains will start appearing. For example:</div><div><br/></div><div class="separator" style="clear: both; text-align: center;"><a href="https://4.bp.blogspot.com/-vyFHhzmswQg/WOdJ1PnoOJI/AAAAAAAAE2E/s7k6FScITP87Kdq4nxTZ-NelS_FsrOgMQCPcB/s1600/Screen%2BShot%2B2017-04-07%2Bat%2B10.11.40.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" height="248" src="https://4.bp.blogspot.com/-vyFHhzmswQg/WOdJ1PnoOJI/AAAAAAAAE2E/s7k6FScITP87Kdq4nxTZ-NelS_FsrOgMQCPcB/s320/Screen%2BShot%2B2017-04-07%2Bat%2B10.11.40.png" width="320"/></a></div><div><br/></div><h2 style="text-align: left;">Analytics Demo</h2><div><br/></div><div><div>The first demo is focused on how you can use Infinispan for doing offline analytics. In particular, this demo tries to answer the following question:</div><div><br/></div><div><i>Q. What is the time of the day when there is the biggest ratio of delayed trains?</i></div></div><div><br/></div><div>Once again, this demo runs on top of OpenShift cloud, uses Infinispan as in-memory data grid for storage and Vert.x for glueing components together.</div><div><br/></div><div><div>To answer this question, Infinispan data grid will be loaded with 3 weeks worth of data from station boards using a Vert.x verticle. Once the data is loaded, the Jupyter notebook will invoke an HTTP restful endpoint which will invoke an Vert.x verticle called <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;"><b>AnalyticsVerticle</b></span>.</div><div><br/></div><div>This verticle will invoke a remote server task which will use <a href="http://infinispan.org/docs/stable/user_guide/user_guide.html#streams">Infinispan Distributed Java Streams</a> to calculate the two pieces of information required to answer the question: per hour, how many trains are going through the system, and out of those, how many are delayed.</div><div><br/></div><div>An important aspect to bear in mind about this server tasks is that it will only be executed in one of the nodes in the cluster. It does not matter which one. In turn, this node will will ship the lambdas required to do the computation to each of the nodes so that they can executed against their local data. The other nodes will reply with the results and the node where the server task was invoked will aggregate the results.</div></div><div><br/></div><div>The results will be sent back to the originating invoker, the Jupyter notebook which will plot the results. The following diagrams shows how the following components interact with each other to achieve this:<br/><br/><div class="separator" style="clear: both; text-align: center;"><a href="https://1.bp.blogspot.com/-0BKwklL6rZE/WQxMl-NCG-I/AAAAAAAAE2k/Z0SZLEhVz4YLIkh3AM8p8EMOW0EN-KMGwCLcB/s1600/analytics-demo.jpg" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" height="354" src="https://1.bp.blogspot.com/-0BKwklL6rZE/WQxMl-NCG-I/AAAAAAAAE2k/Z0SZLEhVz4YLIkh3AM8p8EMOW0EN-KMGwCLcB/s640/analytics-demo.jpg" width="640"/></a></div><br/><br/>Here is the demo step-by-step guide:</div><div><br/></div><div> 1. Start OpenShift Origin and deploy all components as shown in previous demo.</div><div><br/></div><div> 2. Start the Jupyter notebook:</div><div><br/></div><div> <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">cd ~/swiss-transport-datagrid/analytics/analytics-jupyter</span></div><div> <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">~/anaconda/bin/jupyter notebook</span></div><div><br/></div><div> 3.Once the notebook opens, click open live-demo.ipynb notebook and execute each of the cells in order. You should end up seeing a plot like this:</div><div><br/></div><div class="separator" style="clear: both; text-align: center;"><a href="https://4.bp.blogspot.com/-0WwGpCjfU1g/WOdG4rNJ8QI/AAAAAAAAE14/5Lmb3xhx0345zsWn5f_DGI1v-nIEvwzwQCPcB/s1600/Screen%2BShot%2B2017-04-07%2Bat%2B09.58.56.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" height="240" src="https://4.bp.blogspot.com/-0WwGpCjfU1g/WOdG4rNJ8QI/AAAAAAAAE14/5Lmb3xhx0345zsWn5f_DGI1v-nIEvwzwQCPcB/s320/Screen%2BShot%2B2017-04-07%2Bat%2B09.58.56.png" width="320"/></a></div><div><br/></div><div><div>So, the answer to the question:</div><div><br/></div><div><i>Q. What is the time of the day when there is the biggest ratio of delayed trains?</i></div><div><a href="https://www.blogger.com/blogger.g?blogID=5717179571414330874" imageanchor="1" style="clear: right; float: right; margin-bottom: 1em; margin-left: 1em;"/><a href="https://www.blogger.com/blogger.g?blogID=5717179571414330874" imageanchor="1" style="clear: right; float: right; margin-bottom: 1em; margin-left: 1em;"/><a href="https://www.blogger.com/blogger.g?blogID=5717179571414330874" imageanchor="1" style="clear: right; float: right; margin-bottom: 1em; margin-left: 1em;"/><br/></div><div>is <b>2am</b>! That's because last connecting trains of the day wait for each other to avoid leaving passengers stranded.</div></div><div><br/></div><h2 style="text-align: left;">Conclusion</h2><div><br/></div><div><div>This has been a summary of the demos that I presented at<a href="http://www.developermarch.com/developersummit/">Great Indian Developer Summit</a>with the intention of getting you running these demos as quickly as possible. The repository contains more detailed information of these demos. If there's anything unclear or any of the instructions above are not working, please let us know!</div></div><div><br/></div><div>Once again, a very special thanks to Alexandre Masselot for being the inspiration for these demos. Merci @Alex!!</div><div><br/></div><div>Over the next few months we will be enhancing the demo and hopefully we'll be able to do some more live demonstrations at other conferences.</div><div><br/></div><div><div>Cheers,</div><div>Galder</div></div></div></p>
