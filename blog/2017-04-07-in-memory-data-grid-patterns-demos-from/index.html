In Memory Data Grid Patterns Demos from Devoxx France!

<p><div dir="ltr" style="text-align: left;" trbidi="on"><a href="http://devoxx.fr/">Devoxx France 2017</a> was a blast!! Emmanuel and I would like to thank all attendees to our <a href="http://cfp.devoxx.fr/2017/talk/UKK-0693/Architecture_par_la_pratique:patterns_d%E2%80%99utilisation_de_systemes_in-memory-_WD-40_entre_vos_donnees_et_vos_applis">in-memory data grids patterns</a> talk. The room was full and we thoroughly enjoyed the experience!<br/><br/>During the talk we presented a couple of small demos that showcased some in-memory data grid use cases. The demos are located <a href="https://github.com/galderz/datagrid-patterns">here</a>, but I thought it'd be useful to provide some <a href="https://github.com/galderz/datagrid-patterns">step-by-step</a> here so that you can get them running as quickly as possible.<br/><br/>Before we start with any of the demos, it's necessary to run some set up steps:<br/><br/> 1. Check out git repository:<br/><br/><span style="font-family: Courier New, Courier, monospace;"> git clone https://github.com/galderz/datagrid-patterns</span><br/><br/> 2. Download <a href="http://downloads.jboss.org/infinispan/9.0.0.Final/infinispan-server-9.0.0.Final-bin.zip">Infinispan Server 9.0.0.Final</a> and at the same level as the git repository.<br/><br/> 3. Go into the datagrid-patterns directory, start the servers and wait until they've started:<br/><br/><span style="font-family: Courier New, Courier, monospace;"> cd datagrid-patterns</span><br/><span style="font-family: Courier New, Courier, monospace;"> ./run-servers.sh</span><br/><br/> 4. Install <a href="https://www.continuum.io/downloads">Anaconda for Python 3</a>, this is required to run Jupyter notebook for plotting.<br/><br/> 5. Install <a href="https://maven.apache.org/download.cgi">Maven 3</a>.<br/><br/>Once the set up is complete, it's time to start with the individual demos.<br/><br/>Both demos shown below work with the same application domain: rail transport systems. In this domain, we differentiate between physical stations, trains, station boards which are located in stations, and finally stops, which are individual entries in station boards.<br/><br/><h2>Analytics Demo</h2><br/>The first demo is focused on how you can use Infinispan for doing offline analytics. In particular, this demo tries to answer the following question:<br/><br/><i>Q. What is the time of the day when there is the biggest ratio of delayed trains?</i><br/><div><br/></div><div>To answer this question, Infinispan data grid will be loaded with 3 weeks worth of data from station boards. Once the data is loaded, we will execute a remote server task which will use <a href="http://infinispan.org/docs/stable/user_guide/user_guide.html#streams">Infinispan Distributed Java Streams</a> to calculate the two pieces of information required to answer the question: per hour, how many trains are going through the system, and out of those, how many are delayed.</div><div><br/></div><div>An important aspect to bear in mind about this server tasks is that it will only be executed in one of the nodes in the cluster. It does not matter which one. In turn, this node will will ship the lambdas required to do the computation to each of the nodes so that they can executed against their local data. The other nodes will reply with the results and the node where the server task was invoked will aggregate the results.</div><div><br/></div><div>Then, these results are sent back to the client, which in turn, stores the results as JSON in an intermediate cache. Once the results are in place, we will use a Jupyter notebook to read those results and plot the result.</div><div><br/></div><div>Let's see these steps in action:</div><div><br/></div><div> 1. First, we need to install the server tasks in the running servers above:</div><div><br/></div><div><span style="font-family: Courier New, Courier, monospace;"> cd datagrid-patterns/analytics</span></div><div><span style="font-family: Courier New, Courier, monospace;"> mvn clean install package -am -pl analytics-server</span></div><div><span style="font-family: Courier New, Courier, monospace;"> mvn wildfly:deploy -pl analytics-server</span></div><div> </div><div> 2. Open the datagrid-pattern repo with your favourite IDE and run <span style="font-family: Courier New, Courier, monospace;"><b>delays.java.stream.InjectApp</b></span>class located in <span style="font-family: Courier New, Courier, monospace;">analytics/analytics-server</span> project. This command will inject the data into the cache. On my environment, it takes between 1 and 2 minutes.</div><div><br/></div><div> 3. With the data loaded, we need to run the remote task that will calculate the total number of trains per hour and how many of those are delayed. To do that, execute <span style="font-family: Courier New, Courier, monospace;"><b>delays.java.stream.AnalyticsApp</b></span> class located in <span style="font-family: Courier New, Courier, monospace;">analytics/analytics-server</span> project from your IDE.</div><div><br/></div><div> 4. You can verify that the results have been calculating by going to the following address:</div><div><br/></div><div><span style="font-family: Courier New, Courier, monospace;"> <a href="http://localhost:8180/rest/analytics-results/results">http://localhost:8180/rest/analytics-results/results</a></span></div><div><br/></div><div> 5. With the results in place, it's time to start the Jupyter notebook:</div><div><br/></div><div><span style="font-family: Courier New, Courier, monospace;"> cd datagrid-patterns/analytics/analytics-jupyter</span></div><div><span style="font-family: Courier New, Courier, monospace;"> ~/anaconda/bin/jupyter notebook</span></div><div><br/></div><div> 6. Once the notebook opens, click open <span style="font-family: Courier New, Courier, monospace;"><b>live-demo.ipynb</b></span>notebook and execute each of the cells in order. You should end up seeing a plot like this:</div><div><br/></div><div class="separator" style="clear: both; text-align: center;"><a href="https://1.bp.blogspot.com/-0WwGpCjfU1g/WOdG4rNJ8QI/AAAAAAAAE10/L0bV9eq8RZgIDw-C51UyukbZRuZJUyPCQCLcB/s1600/Screen%2BShot%2B2017-04-07%2Bat%2B09.58.56.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" height="240" src="https://1.bp.blogspot.com/-0WwGpCjfU1g/WOdG4rNJ8QI/AAAAAAAAE10/L0bV9eq8RZgIDw-C51UyukbZRuZJUyPCQCLcB/s320/Screen%2BShot%2B2017-04-07%2Bat%2B09.58.56.png" style="cursor: move;" width="320"/></a></div><div class="separator" style="clear: both; text-align: center;"><br/></div><div class="separator" style="clear: both; text-align: left;">So, the answer to the question:</div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;"><i>Q. What is the time of the day when there is the biggest ratio of delayed trains?</i></div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;">is <b>2am</b>! That's because last connecting trains of the day wait for each other to avoid leaving passengers stranded.</div><div class="separator" style="clear: both; text-align: left;"><br/></div><h2>Real Time Demo</h2><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;">The second demo that we presented uses the same application domain as above, but this time we're trying to use our data grid as a way of storing the station board state of each station at a given point in time. So, the idea is to use Infinispan as an in memory data grids for working with real time data.</div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;">So, what can we do with this type of data? In our demo, we will create a centralised dashboard of delayed trains around the country. To do that, we will take advantage of <a href="http://infinispan.org/docs/stable/user_guide/user_guide.html#query.continuous">Infinispan's Continuous Query</a> functionality which allows us to find those station boards which contain stops that are delayed, and as new delayed trains appeared these will be pushed to our dashboard.</div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;">To run this demo, keep the same servers running as above and do the following:</div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;">1. Run <span style="font-family: Courier New, Courier, monospace;"><b>delays.query.continuous.FxApp</b></span> application located in <span style="font-family: Courier New, Courier, monospace;">real-time</span> project inside the datagrid-patterns demo. This app will inject some live station board data and will launch a JavaFX dashboard that shows delayed trains as they appear. It should look something like this:</div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: center;"><a href="https://2.bp.blogspot.com/-vyFHhzmswQg/WOdJ1PnoOJI/AAAAAAAAE2A/Ui1YS1T93JIxPKpwbS6u0p1whsok3nxfwCLcB/s1600/Screen%2BShot%2B2017-04-07%2Bat%2B10.11.40.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" height="248" src="https://2.bp.blogspot.com/-vyFHhzmswQg/WOdJ1PnoOJI/AAAAAAAAE2A/Ui1YS1T93JIxPKpwbS6u0p1whsok3nxfwCLcB/s320/Screen%2BShot%2B2017-04-07%2Bat%2B10.11.40.png" width="320"/></a></div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;"><br/></div><h2 style="text-align: left;">Conclusion</h2><div class="separator" style="clear: both; text-align: left;">This has been a summary of the demos that we run in our talk at Devoxx France with the intention of getting you running these demos as quickly as possible. The repository contains more detailed information of these demos. If there's anything unclear or any of the instructions above are not working, please let us know!</div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;">Thanks to Emmanuel Bernard for partnering with me for this Devoxx France talk and for the continuous feedback while developing the demos. Thanks as well to Tristan Tarrant for the input in the demos and many thanks to all Devoxx France attendees who attended our talk :)</div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;">A very special thanks to<b>Alexandre Masselot</b> whose <a href="https://www.slideshare.net/alexmass/swiss-transport-in-real-time-tribulations-in-the-big-data-stack">"Swiss Transport in Real Time: Tribulations in the Big</a> Data Stack" talk at Soft-Shake 2016 was the inspiration for these demos. @Alex, thanks a lot for sharing the demos and data with me and the rest of the community!!</div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;">In a just a few weeks I'll be at <a href="http://www.developermarch.com/developersummit/">Great Indian Developer Summit</a> presenting these demos and much more! Stay tuned :)</div><div class="separator" style="clear: both; text-align: left;"><br/></div><div class="separator" style="clear: both; text-align: left;">Cheers,</div><div class="separator" style="clear: both; text-align: left;">Galder</div></div></p>
