DevNation Live Blog: Building Reactive Applications with Node.js and Red Hat JBoss Data Grid

<p><div dir="ltr" style="text-align: left;" trbidi="on">Last Tuesday I gave a talk at DevNation 2016 onbuilding Reactive Applications with Node.js and Red Hat JBoss Data Grid. The slides for it will be uploaded to the DevNation site shortly, but for those who want to play around with the application demoed in the talk, you can find the code and instructions in <a href="https://github.com/galderz/infinispan-events/tree/june16">this repository</a>.<br/><br/>The Red Hat Developers blog posted a review of the talk in this address:<a href="http://developers.redhat.com/blog/2016/06/29/devnation-live-blog-building-reactive-applications-with-node-js-and-red-hat-jboss-data-grid/">http://developers.redhat.com/blog/2016/06/29/devnation-live-blog-building-reactive-applications-with-node-js-and-red-hat-jboss-data-grid/</a><br/><br/>Please find below review as written by Rob Terzi:<br/><br/><h1 class="entry-title" style="-webkit-font-smoothing: antialiased; box-sizing: border-box; color: #646464; font-style: italic; margin: 0px; padding: 0px 0px 15px;">Posted by<a href="http://developers.redhat.com/blog/author/rctkv/" style="-webkit-font-smoothing: antialiased; -webkit-hyphens: auto; -webkit-transition: all 0.3s ease 0s; box-sizing: border-box; color: #333333; cursor: pointer; line-height: inherit; overflow-wrap: break-word; text-decoration: none; transition: all 0.3s ease 0s; word-break: break-word; word-wrap: break-word;">Rob Terzi</a>on<time class="entry-date published" datetime="2016-06-29T11:00:43+00:00" style="-webkit-font-smoothing: antialiased; box-sizing: border-box;">June 29, 2016</time></div></div><div style="-webkit-font-smoothing: antialiased; box-sizing: border-box; color: #646464; font-family: overpass, Arial, sans-serif; font-size: 14px; line-height: 1.5; margin-bottom: 10px; padding: 0px; text-rendering: optimizelegibility;">At DevNation, Red Hat&#x2019;sGalder Zamarre&#xF1;o gave a talk with a live demo,Building reactive applications with Node.js and Red Hat JBoss Data Grid. The demo consisted ofbuilding an event-based three tier web application using JBoss Data Grid (JDG) as the data layer, an event manager running on Node.js, and a web client. Recently, support for Node.js clients was added to JDG, opening up the performance of a horizontally scalable in-memory data grid, to reactive web and mobile applications.</div><div style="-webkit-font-smoothing: antialiased; box-sizing: border-box; color: #646464; font-family: overpass, Arial, sans-serif; font-size: 14px; line-height: 1.5; margin-bottom: 10px; padding: 0px; text-rendering: optimizelegibility;">JDG is capable of processing and storing real-time streams of data, while maintaining very fast response times. It does this by using the memory available from a dynamically scalable grid of machines. Galder described JDG as a four-in-one package capable of being:</div><ul style="-webkit-font-smoothing: antialiased; box-sizing: border-box; color: #646464; font-family: overpass, Arial, sans-serif; font-size: 14px; line-height: 1.5; list-style-position: outside; margin: 0px 0px 10px 1.1rem; padding: 0px;"><li style="-webkit-font-smoothing: antialiased; box-sizing: border-box; margin: 0px 0px 0px 1em; padding: 0px;">a distributed cache.</li><li style="-webkit-font-smoothing: antialiased; box-sizing: border-box; margin: 0px 0px 0px 1em; padding: 0px;">a high performance NoSQL primary data store.</li><li style="-webkit-font-smoothing: antialiased; box-sizing: border-box; margin: 0px 0px 0px 1em; padding: 0px;">an event-driven data store,particularly for real time event processing.</li><li style="-webkit-font-smoothing: antialiased; box-sizing: border-box; margin: 0px 0px 0px 1em; padding: 0px;">a big data and Internet of Things (IoT) data store.</li></ul><div style="-webkit-font-smoothing: antialiased; box-sizing: border-box; color: #646464; font-family: overpass, Arial, sans-serif; font-size: 14px; line-height: 1.5; margin-bottom: 10px; padding: 0px; text-rendering: optimizelegibility;">The three-tiered web app in the demo consisted of:</div><ul style="-webkit-font-smoothing: antialiased; box-sizing: border-box; color: #646464; font-family: overpass, Arial, sans-serif; font-size: 14px; line-height: 1.5; list-style-position: outside; margin: 0px 0px 10px 1.1rem; padding: 0px;"><li style="-webkit-font-smoothing: antialiased; box-sizing: border-box; margin: 0px 0px 0px 1em; padding: 0px;">A web client written in Elm, which is a functional language that compiles to JavaScript. It is statically typed, which the presenter feels leads to well architected code. Elm competes with platforms such asReact and Angular. Any of those other platforms could be used, but Galder chose Elm for the live demo, particularlygiven the useful error messages the compiler generates as a virtue of using astatically typed language.</li><li style="-webkit-font-smoothing: antialiased; box-sizing: border-box; margin: 0px 0px 0px 1em; padding: 0px;">An event manager running on Node.js using Express.js.</li><li style="-webkit-font-smoothing: antialiased; box-sizing: border-box; margin: 0px 0px 0px 1em; padding: 0px;">JBoss Data Grid as the data store. Three nodes were used, running on the same laptop. Each element was guaranteed to be stored in two nodes, providing redundancy for fail over.</li></ul><div style="-webkit-font-smoothing: antialiased; box-sizing: border-box; color: #646464; font-family: overpass, Arial, sans-serif; font-size: 14px; line-height: 1.5; margin-bottom: 10px; padding: 0px; text-rendering: optimizelegibility;">Node.js based applications have become very popular. Many use JavaScript on all three tiers, including NoSQL data stores. However, most of those data stores can&#x2019;t match the scalability and response times of JDG. Traditionally, developers have needed to use Java to take advantage of JBoss Data Grid. The new fully asynchronous Node.js interface to JBoss Data Grid should enable developers to build some truly interesting next-generation reactive applications.</div><div style="-webkit-font-smoothing: antialiased; box-sizing: border-box; color: #646464; font-family: overpass, Arial, sans-serif; font-size: 14px; line-height: 1.5; margin-bottom: 10px; padding: 0px; text-rendering: optimizelegibility;">You can download JBoss Data Grid from<a href="http://developers.redhat.com/downloads/" style="-webkit-font-smoothing: antialiased; -webkit-hyphens: auto; -webkit-transition: all 0.3s ease 0s; box-sizing: border-box; color: #08c0fc; cursor: pointer; line-height: inherit; overflow-wrap: break-word; text-decoration: none; transition: all 0.3s ease 0s; word-break: break-word; word-wrap: break-word;">developers.redhat.com</a>. If you&#x2019;d like to get involved, join the open source community at<a href="http://infinispan.org/" style="-webkit-font-smoothing: antialiased; -webkit-hyphens: auto; -webkit-transition: all 0.3s ease 0s; box-sizing: border-box; color: #08c0fc; cursor: pointer; line-height: inherit; overflow-wrap: break-word; text-decoration: none; transition: all 0.3s ease 0s; word-break: break-word; word-wrap: break-word;">infinispan.org</a>.</div></div></p>
